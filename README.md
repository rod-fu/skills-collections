# public-sector-job-search 


核心问题：

给定一份简历，哪些岗位真正值得投递？


背景：

体制内招聘信息分散，同一候选人需要同时关注多个渠道。招聘平台虽能发现线索，但关键信息仍需回到官方公告确认：信息来源是否可靠、报名是否有效、条件是否匹配、岗位性质是否明确等。


核心能力：

简历画像提取 — 学历、专业、经历、技能、地域偏好等
官方来源优先 — 以官方公告为准，第三方平台仅作线索
报名资格筛查 — 核对截止时间、年龄、学历、专业、工作年限、证书等硬性条件
岗位排序与提示 — 综合匹配度、相关性、平台价值、截止时间、准备成本等因素排序，并标注风险点


使用示例：

"根据我的简历，帮我找适合的岗位。"
"我想留在北京，金融硕士，2026届，党员，有投行和固收实习经历。帮我筛选目标岗位。"


输出格式：

结构化表格（优先级、单位、性质、岗位、地点、截止时间、匹配度、风险点、官方链接）+ 推荐理由、需确认事项、投递顺序、准备材料等。


安装方式：

将 skill 文件夹复制到 Codex skills 目录，运行校验脚本确认安装成功。


注意事项：

处理简历时注意保护个人隐私，删除敏感信息
所有信息以官方公告和报名入口为准
招聘信息时效性强，建议优先联网核验最新信息
岗位性质、编制情况需人工复核，不宜断言


# Word to PPT Extractor

`word-to-ppt-extractor` 是一个面向 Codex 的演示文稿生成 Skill，用于将 Word 文档、`.docx` 文件或长篇文本提炼为结构化 PPT 素材，并进一步生成演示大纲、逐页视觉方案、HTML 预览、PDF，或在需要时导出可编辑的 PPTX。

它的核心不是简单搬运 Word 内容，而是先重构信息密度：识别文档主题、核心论点、受众目标和一句话主张，再按照“一页一个观点”的原则生成适合演讲和汇报的幻灯片结构。

Features

- 从 `.docx` 或长文本中提取标题、正文、表格和图片线索
- 自动生成结构化演示大纲和逐页内容设计
- 支持大胆 / 中性 / 安静三类视觉风格方向
- 可输出 HTML 3D 概览演示、高保真 PDF
- 可在明确需要时生成基础可编辑 PPTX
- 内置页面标题、信息密度、视觉母题和交付格式约束

Use Cases

适用于将报告、方案文档、访谈稿、研究材料、项目汇报、培训材料等内容转化为更适合展示的演示文稿素材。

Design Philosophy

好的 PPT 不是 Word 的分页版，而是对复杂信息的重新组织。  
这个 Skill 强调观点先行、受众对齐、视觉转译和多格式交付，帮助用户从长文档中快速得到可讲、可看、可交付的演示内容。
