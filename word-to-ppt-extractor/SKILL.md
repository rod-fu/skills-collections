# Word 转 PPT 提炼器

## 核心原则

将 Word 转成 PPT 时，先做信息密度的重构，而不是格式搬运。始终按“目的与受众 -> 结构摘要 -> 观点化大纲 -> 单页视觉转译 -> 多通道交付”的顺序推进。

必须遵守：
- 先确立演示目的、目标受众和一句话金句，再处理具体内容。
- PPT 总内容量控制在原文的 10% 到 15%。
- 每页正文要点不超过 5 条，每条原则上不超过 15 个字。
- 页面标题必须是明确判断句或核心主张，禁止使用“市场分析”“项目背景”这类目录词。
- 拒绝“这里放一张配图”等通用占位，必须给出具体视觉母题、图解结构和版式方案。
- 若文档涉及 2024 年以后时间线、产品、技术或规格，先联网验证最新事实；若不能联网，明确说明未验证状态。
- 若文档涉及具名品牌，先询问/查找品牌规范，使用真实 logo、产品图和真实色值，并固化为 `brand-spec.md`。

## 快速决策树

1. 用户只要“大纲/素材”：输出结构化大纲和逐页视觉方案，不生成文件。
2. 用户要“HTML/网页演示/预览”：生成多页 HTML 和 `index.html` 3D 概览墙。
3. 用户要“PDF”：先生成 HTML，再导出 PDF；如果 PDF 工具链不可用，交付 HTML 并说明 PDF 未导出原因。
4. 用户要“PPT/PowerPoint/Keynote 可编辑”：先读取 `references/editable-pptx.md`，生成可编辑优先的 PPTX；复杂视觉必须降级为基础形状和文本。
5. 用户同时要 HTML 与 PPTX：HTML 作为高保真版本，PPTX 作为可编辑版本；输出时明确两者差异。

## 🔴 CHECKPOINT

在以下情况暂停并让用户确认，不要自作主张继续：
- 无法判断演示目的、受众和一句话金句，且文档也无法推断。
- 文档涉及具名品牌，但没有品牌规范、真实 logo 或产品图来源。
- 用户要求 PPTX，同时又要求复杂 CSS、动画或 3D 效果完全可编辑。
- 需要联网验证 2024 年以后事实，但当前环境无法联网。

## 使用流程

### 1. 采集取舍标尺

向用户确认或从文档中推断以下信息：
- 演示目的：说服决策者、同步进展、方案提案、培训说明或其他。
- 目标受众：背景、关注点、痛点、决策权重。
- 一句话金句：听众只能记住一句时，应记住什么。
- 建议页数、交付形式和是否需要可编辑 PPTX。
- 视觉温度：大胆、中性、安静。若用户模糊，运行 `scripts/style_picker.py` 生成“秒数轮盘”方案，并同时给出现实参照和最佳设计师方案。

如果用户没有回答，按文档类型推断：
- 政务、科研、技术设计文档：默认“安静”或“中性”。
- 路演、发布、对外提案：默认“大胆”。
- 管理汇报、咨询方案、数据分析：默认“中性”。

### 2. 解析 Word 或长文

优先使用 `scripts/extract_docx_text.py` 提取 `.docx` 文本、标题层级、表格文本和图片清单。纯文本输入则直接分析。

先输出“读懂与对齐”摘要，不要立刻生成幻灯片：
- 文档主题。
- 核心论点。
- 关键证据、数据、案例。
- 可能的缺口或事实待验证点。

### 3. 生成观点化大纲

使用“一页一观点”原则重写结构：
- 结论层：10 到 15 字内的判定句，作为页面大标题。
- 支撑层：2 到 3 条核心论据，每条不超过 15 字。
- 细节层：推导、背景和补充说明进入演讲备注。

若核心论点缺失，从前言、摘要或结论中提出 3 个候选主线，供用户选择，不能生成无主线素材包。

### 4. 映射视觉风格

读取 `references/style-library.md`，按视觉温度选择 PPT 风格库：
- 大胆：1 到 8 号。
- 中性：9 到 14 号。
- 安静：15 到 20 号。

识别每页的信息关系并映射图解：
- 步骤/因果：流程箭头、左右大字流、阶段墙。
- 对比/优劣：Bento 对称分割、咨询版细线分割。
- 数据/趋势：巨型数字、Sparkline、极简图表。
- 理论框架：Golden Circle、同心圆、三角几何。

每页必须写出色彩推导：“采样 -> 收敛 -> 论证”，说明主色取自何处以及为什么。

### 5. 生成交付物

默认生成 HTML 3D 概览演示：
1. 为每页创建一个独立 HTML 文件。
2. 复制 `assets/deck_index.html` 为输出目录的 `index.html`。
3. 将 `MANIFEST` 替换为页面清单。
4. 需要动效时参考 `assets/animations.jsx` 的 Stage/Sprite/useTime 结构。

自动伴生导出 PDF：
- 使用 `scripts/export_deck_pdf.mjs` 从 `index.html` 或逐页 HTML 导出 PDF。
- PDF 应作为高保真兜底交付物。

仅当用户明确要求可编辑 PowerPoint/Keynote 时生成 PPTX：
- 先读取 `references/editable-pptx.md`。
- 使用 `scripts/export_deck_pptx.mjs`。
- 若复杂 CSS 降级，附带“排版降级说明”。

### 6. 验证交付物

生成文件后必须验证：
- HTML：检查 `index.html`、每页 HTML、`MANIFEST` 引用和首屏标题是否存在。
- PDF：确认文件存在、页数合理、文字没有明显裁切。
- PPTX：渲染预览或运行可用的溢出检查；若工具链不可用，至少检查文件存在、页数、文本对象和备注。
- 脚本失败时记录具体错误，不要只说“生成失败”。

## 输出格式

必须输出以下结构：

```markdown
# PPT 主题：[基于核心论点生成的标题]
## 视觉风格：[PPT 风格库具体名称] | 视觉温度：[大胆/中性/安静]
## 色彩推导：[采样 -> 收敛 -> 论证] | 字体配对：[Display 标题 + Body 正文]
## 建议总页数：[X] 页 | 交付形式：[HTML 3D 概览墙 / PDF / PPTX]
## 演示哲学对准仪：
- **核心目的**：[...]
- **目标受众**：[...]
- **一句话金句**： [...]

### 目录/结构大纲
1. 模块一：[模块标题]（建议分配 X 页）
2. 模块二：[模块标题]（建议分配 Y 页）

### 逐页素材设计详情

#### Page [N]：[明确结论判定句]
- **视觉母题**：[内容独有的视觉种子/结构隐喻]
- **视觉版式方案**：[对标选定风格库的具体布局]
- **正文要点（支撑层）**：
  * [15 字内要点]
- **页面核心视觉/图表**：[精确图解或 CSS Grid 格局]
- **演讲备注（细节层）**：[推导、背景、宣讲口语]

### 交付物链接/预览
- **[在线预览并编辑 3D 概览墙 HTML]**
- **[下载高保真矢量 PDF]**
- **[下载可编辑 PPTX 文件]**（若适用）
```

## 资源导航

- `scripts/extract_docx_text.py`：提取 `.docx` 的文本、标题、表格和图片清单。
- `scripts/style_picker.py`：按秒数轮盘和视觉温度推荐风格。
- `scripts/scaffold_deck.py`：根据 JSON 大纲生成多页 HTML 和 `index.html`。
- `scripts/export_deck_pdf.mjs`：用 Playwright 导出 PDF。
- `scripts/export_deck_pptx.mjs`：生成基础可编辑 PPTX。
- `references/style-library.md`：40 种 HTML/PPT 原生风格库。
- `references/editable-pptx.md`：可编辑 PPTX 的 4 条硬约束。
- `references/output-contract.md`：视觉合规大纲与失败处理细则。

## 失败模式速查

| 触发条件 | 一线修复 | 仍失败兜底 |
|---|---|---|
| `.docx` 解析失败 | 检查路径、文件是否被占用，改用 `extract_docx_text.py` 重新解析 | 请求用户另存为 `.docx` 或粘贴正文 |
| 文档没有清晰主线 | 从前言、摘要、结论提取 3 个候选主线 | 暂停让用户选择主线 |
| HTML 预览空白 | 检查 `MANIFEST`、页面路径和 iframe `src` | 交付逐页 HTML 并说明概览墙待修复 |
| PDF 导出缺少 Playwright | 改用 HTML 作为高保真交付物 | 提示安装/加载 Playwright 后再导出 PDF |
| PPTX 样式错位 | 按 `editable-pptx.md` 降级为基础形状 | 输出“排版降级说明”，建议以 HTML/PDF 展示 |
| 品牌资产不可得 | 请求用户提供规范或授权检索 | 使用中性无品牌方案，不伪造 logo |

## 反例黑名单

不要做：
- 不要把 Word 段落直接搬到幻灯片上。
- 不要用“背景介绍、市场分析、解决方案”作为页面标题。
- 不要用“这里放一张配图”作为视觉方案。
- 不要用通用图标替代真实品牌资产。
- 不要在 PPTX 中承诺复杂 CSS、动画、3D 效果完全可编辑。
- 不要在没有事实验证时把 2024 年以后的产品、标准、规格写成确定结论。
- 不要为了显得完整而超过每页 5 条要点。
